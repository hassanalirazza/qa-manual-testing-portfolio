# QA Manual Testing Portfolio

A collection of structured manual testing artifacts demonstrating test case design, test planning, requirement traceability, and quality checklists.

All examples use **fictional applications** and original content — no proprietary or confidential material.

---

## What's Inside

| Folder | Contents |
|---|---|
| [`test-cases/`](./test-cases) | Detailed functional test cases for common application flows |
| [`test-plans/`](./test-plans) | A sample test plan following industry structure |
| [`checklists/`](./checklists) | Smoke and regression testing checklists |
| [`rtm/`](./rtm) | A Requirement Traceability Matrix mapping requirements to test cases |
| [`templates/`](./templates) | Reusable test case template |

---

## Sample Application: ShopEase

To keep examples consistent, most artifacts are written against a fictional e-commerce web application called **ShopEase**, featuring user login, product search, cart, and checkout flows.

---

## Testing Approach

These artifacts follow the **STLC (Software Testing Life Cycle)**:

```
Requirement Analysis → Test Planning → Test Case Design
   → Test Execution → Defect Reporting → Test Closure
```

Each test case includes a unique ID, preconditions, steps, test data, expected results, and priority — written to be executable by any tester without additional context.
